# 选择性委派契约

仅当 Codex 协作工具可用，且工作边界明确、相互独立、确实能从独立上下文或并行执行中获益时才委派。步骤很小、严格串行、与当前改动紧耦合或容易产生文件冲突时，由主 agent 本地完成。不要给每个琐碎动作都创建一个 subagent。

## 交给 delegate 什么

- **范围**——EXEC_PLAN 中一个内聚目标及其绑定检查。明确写：「只做这个范围，不要继续后续动作。」
- **文件**——允许新建或修改的确切文件/模块。并发 delegate 的文件所有权不得重叠；要求遵循 `AGENTS.md` 做 surgical changes。
- **上下文**——相关的原始计划章节、差距清单事实和现有接口。若委派用于独立验证，不要泄露预期答案。
- **运行环境**——使用 `.env` 的 `CONDA_HOME` / `PYTHON_HOME`；绝不用系统 Python，也不硬编码本地路径。
- **边界**——只跑轻量验证。若动作位于 STOP 线，只准备命令并返回，不得执行。
- **产出**——`wkdrs/<run>/` 下的确切产物路径。

## 必须返回的结构

- `changed`——新建或修改的文件，每个一行。
- `ran`——实际运行的命令及结果，或 `none`。
- `check`——`pass` / `fail` 加证据。
- `blockers`——未解决的阻塞，或 `none`。
- `handoff`——准备好的 STOP 线命令，或 `none`。

## 主 agent 的责任

主 agent 必须审查 diff、解决集成问题，并在写入 EXEC_LOG 前重跑或独立检查绑定验证。delegate 自报的通过不能直接作为最终证据。

- 通过 → 记录产物与检查证据，将动作标为 `done`。
- 失败 → 只有存在具体修正方案时才重试，最多两次；否则标 `blocked` 并停止。
- 有 handoff → 记入「待用户执行」，不得运行，随后停止。
