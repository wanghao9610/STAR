# aggregate 模式——跨 run 的结果总账

单 run 分析回答*这个 run 是否达到了它的计划*。aggregate 回答*整个实验计划显示了什么、每个数字从哪来*——把每个 run 已验证的数字编译成一份总账 `metds/results.md`：论文结果节据以撰写的素材，也是 `metds/evaluation.md` 的对应物——后者定义这些数字是在什么协议下测出来的。

本文件定义范围、信任模型、编排轴、总账绝不做的事、排除，以及写入门。

## 范围

`aggregate` 不带计划名时覆盖整片森林；带计划名时覆盖该节点的子树。收集范围内每个叶子，以及**它 `exec_runs` 里的每一个 run**——一个叶子若为换 seed 或修 bug 重跑过，就有好几个；总账呈现的是实际存在的东西，而不只是最后一个。逐 run 取首个匹配：

- 该 run 目录里最新的 `wkdrs/<run>/EXPT_ANALYSIS_<日期>.md` → 一个可汇总的 run，占表里一行；
- run 目录没有分析报告 → **缺口**：列出它，路由到 `star-expt-analyst <run 目录>`。不要在这里读它的日志——aggregate 读的是报告，不是原始 run；
- 叶子压根没有 `exec_runs` → 从未执行；列为缺口，路由到 `star-plan-executor <slug>`。

范围内没有任何叶子有报告，是一个合法答案：如实说明并停止。绝不凭空编译总账。

## 信任模型:回到源头复核,绝不转手相信

一份 EXPT_ANALYSIS 报告是**已验证**的产物——里面每个数字入报告前都被在其引用的来源处重开确认过（核心原则 2）。但那不是照抄它的许可：报告写完后 run 可能已经变了，而总账里的数字正是会被抄进论文的那个。

对每个将要进入 `results.md` 的数字：

1. 从报告的指标表里读出它，连同记录的来源（`path:line` 或键名）与 split。
2. **重开那个来源**，确认它仍然说的是报告所说的。
3. 一致 → 该数字进入总账，携带 `{run、来源、报告日期}`。
4. 不一致，或来源已不存在 → 该数字**不**进入。把该 run 放进 §5 已排除，写明两个值与原因，并建议重跑 `star-expt-analyst <slug>` 刷新那份报告。

绝不从任何报告未覆盖的原始日志里读指标。那是单 run 分析，属于带自己 D 维验证的单 run 流程——不属于这里。

## 按主张编排,而非按 run 编排

计划树是按拆解组织的；读者需要的是按**数字显示了什么**来组织。从根计划 §4 分组：

- **claim → 实验映射**给出每条 claim / benchmark 一张表；
- **消融设计**给出每个消融一张表，行是各变体；
- 设计没有映射到的 run 进最后的"其他 run"块——列出来，绝不悄悄丢掉。

表内的行是 §4 指派给它的那些 run，按 §4 给出的顺序（没有则按前缀）。一个 run 一行，不是一份报告一行。

## 总账绝不做的事

- **绝不把差值归因于某个原因。** 表显示的是变体 → 数字。说清某个变体*为什么*赢，需要一次这个 skill 并不运行的受控对比（核心原则 5）。根 §4 说某变体*测试*什么，那是表题；在这里下的结论不是。
- **绝不合并不同协议下测出的数字。** split 不同、指标定义不同、评测入口不同，就是不同的表——或者一张表加 `⚠` 并点名两种协议。悄悄并成一列，就是一张错的论文表。
- **绝不算出一个没人测过的 baseline 差值。** 计划写明的 baseline 数字是一行；没测过的是 `not measured`，不是推算出来的差值。
- **绝不复述协议。** benchmark 怎么跑归 `metds/evaluation.md`；总账引用它、只呈现分数。

## 已排除的 run

报告判定为 `invalid` 或 `inconclusive` 的 run，或其数字复核未通过的 run，**不**出现在结果表里。它进 §5 已排除，附判定与一行原因。悄悄排除一个 run，正是一张结果表开始撒谎的方式：读者必须能看见被拿掉了什么、并数得出来。

**`not met` 的 run 不排除**——负结果也是结果，它带着自己的判定留在它该在的表里。

## Frontmatter 契约

```yaml
---
type: results
language: <en|zh>
generated: <YYYY-MM-DD>          # 真实日期；绝不编造
scope: <这些数字来自哪个子树，或 "whole forest">
sources:                          # 喂给这份总账的每一个 run
  - run: <prefix>_<slug>
    report: EXPT_ANALYSIS_<YYYY-MM-DD>.md
    verdict: <met|partially met|not met|inconclusive|invalid>
---
```

`type: results` + `generated:` 正是把文件标记为"编译产物、因而可安全重生成"的东西。`sources:` 是陈旧检查：某个 run 的最新报告比这里记录的更新，就说明总账陈旧了。

## 写入门

- **不存在** → 写。
- **存在且带 `type: results`** → 与新编译内容比对，展示变更清单（每张表一行：`added` / `rewritten` / `removed` / `unchanged`，以及变了什么），再问覆盖还是跳过。所有表都 `unchanged` → 什么都不写；文件与它的 `generated` 日期原样不动。
- **存在但没有该 frontmatter** → 人工撰写的。说清它现在有什么、编译会用什么替换它，然后询问。放着不动是合法结果；编译到用户指定的另一个路径也是。
