# Refs Rubric — 一次合格的文献综述要满足什么

打分对象有两个：逐篇分析笔记，以及 `reference.bib` 及其 index。红线不可谈判；质量标准则决定这是一个文献基座还是一份标题清单。

## 红线（违反即本轮作废）

1. 每个 bib 字段都能追溯到本次运行抓回并缓存在 run 目录里的记录（`references/source_policy_zh.md`）。没有凭记忆的字段，没有推断的字段，没有"明显应该是这个"的填充。
2. 每个 citekey 在 `refs_index.md` 里都有 provenance 行：来源、记录 URL、抓取日期。
3. 每一条都能用一句话说清它与本方法主题的关系。为凑数塞进关系松散的工作，比如实报 43 条更糟。
4. 每篇核心论文都有分析笔记；每篇笔记的 citekey 都在 `reference.bib` 里。
5. 除 `metds/refs/**` 与 `wkdrs/refs_<date>/**` 外不写任何文件。

## 核心论文的挑选（5–10 篇）

一篇论文靠**近**拿到笔记，不是靠有名：

- **直接重叠** —— 同一个任务，或同一机制用在别处。一篇 2019 年正好在做本方法这件事的论文，胜过一篇 2024 年顺带提了一句的综述。
- **定位价值** —— 本方法必须与之划清界限的工作；审稿人一定会问到的那篇。
- **baseline / 基准地位** —— 实验将要对比的对象。
- **新旧** —— 相关性相当时的 tie-break，绝不是主轴。

每条候选都带一句话理由。一句话说不清为什么要读它的，就不是核心论文。

## 分析笔记

每篇笔记（`assets/ref_analysis_template_zh.md`）按这些标准打分：

- **具体胜过含糊。** "在 400M 图文对上做对比学习"——不是"利用了大规模数据"。数字来自论文，数据集点名，baseline 点名。
- **方法可复述。** 没读过原文的人读完能说出它做了什么、为什么应该有效。只写承重的公式，符号要定义。
- **与本项目的关系才是重点。** 泛泛的摘要网上到处都是；笔记值钱在 §5——共同点、差异点、可借鉴、以及它让本项目能立什么。一个换个项目也照样成立的 §5，就是失败的 §5。
- **对读到什么程度诚实。** 至少读摘要、intro、方法和主结果表。若只够到摘要和 intro，`depth:` 就如实写——承认自己浅的笔记有用，装深的没用。
- **claim 是论文的，不是你的。** 数字照论文报（指标 + 数据集 + split）。绝不外推，绝不跨不可比设定做比较，不确定的标 `[unverified]`。

## reference.bib 的组织

- **3–8 个类别。** 少于 3 类算不上分类；多于 8 类把领域切碎了。
- 类别从实际收上来的东西的语义里长出来——标题、任务、机制——不是事先强加的分类体系。
- 命名要贴这批文献：`Open-vocabulary detection`，不是 `Related methods`。
- 每类一个 `%%` 块头：类名、条目数、一行范围说明。组内按年份升序、再按 citekey 排序。
- 每条恰好归入一类。确实两头不靠的进最后的 `Other / cross-cutting` 块，占比封顶约 10%——超了说明类别划错了，不是论文有问题。

## refs_index.md

- provenance 表覆盖 100% 的条目。
- 类别表的计数之和等于条目总数。
- 待人工核对清单即使为空也要在（写"无"）——整节缺失读起来就像"什么都没出问题"。
- 自拟缩写（†）与 arXiv-only（‡）要标出来。

## 聊天摘要

约 400 字内：方法来源、写了哪些笔记、条目数与各类别计数、自审计结果、需要人工处理的部分、下一个 skill。数要报实——缺口就按缺口报。
